# Contributing to Public CLIs

Thank you for your interest in contributing! This project is a curated list of CLI tools, and we welcome additions and improvements.

## How to Contribute

### Adding a CLI

1. Fork this repository
2. Add the CLI entry to the appropriate category in `README.md`
3. Follow the format below:

```markdown
| [CLI Name](https://github.com/owner/repo) | Short description | Platform | `install command` | ![Stars](https://img.shields.io/github/stars/owner/repo.svg?style=social&label=) |
```

4. Submit a Pull Request

### Guidelines

- **One CLI per Pull Request** for easy review
- **CLI must be publicly available** (open-source or free tier)
- **Keep descriptions concise** — under 100 characters
- **Alphabetical order** within each category
- **No duplicates** — check if the CLI is already listed
- **Working install command** — verify the install command works

### Format

| Field | Description |
|:------|:-----------|
| CLI | Name with link to repo or homepage |
| Description | What the CLI does in a short sentence |
| Platform | Supported operating systems |
| Install | Install command or link |
| Stars | GitHub stars badge (if applicable) |

### Categories

If your CLI doesn't fit any existing category, feel free to propose a new one. Just make sure to:

1. Add it to the Index
2. Create the category section with the standard table header
3. Add the "Back to Index" link

## Code of Conduct

Be respectful and constructive. We're all here to build a useful resource for the community.
